# Archived Commerce Docs

This directory contains **historical** material that is kept for reference only.

## What belongs here

- completed implementation plans
- historical architecture visions
- one-off audit reports
- raw test run captures
- historical upgrade notes
- legacy development notes that no longer match the current repo layout

## What does not belong here

- current package behavior
- current configuration guidance
- current installation or upgrade instructions
- current admin or runtime workflows

## Source-of-truth reminder

For current behavior, prefer:

1. `docs/index.md`
2. `packages/*/docs/*.md`
3. active root docs under `docs/`

Only use archived files when you explicitly need historical context.
