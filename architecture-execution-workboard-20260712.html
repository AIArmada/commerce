Critical algorithm sketches

Inventory command path

Order transition commits
  → one semantic Order event after commit
  → one translator listener
  → Inventory command(operation_key)
  → lock/create InventoryOperation
  → if completed: return prior result
  → mutate stock/movements once
  → mark completed in same transaction

Illustrative control flow, not a final interface. The approved design record controls names and types.

Cross-contract review

field | producer | consumer | type | invariant
correlation | Checkout | all modules | approved value
money | Pricing/Discount | Order | minor units + currency
line | Checkout | Order/Inventory | stable line identity
operation | each module | retry caller | durable unique key
outcome | module | Finalizer | stable observable result

CTR-620 must be approved before package implementation.

Checkout finalization

materialize Order
  → confirm/register payment if required
  → commit Discounts
  → commit Inventory
  → record all required phases
  → transition Checkout Completed once
  → emit CheckoutCompleted once
  → clear cart last

Failures remain recoverable and retries resume from durable phase evidence.

Shipment remote operation

persist operation intent + idempotency key
  → call carrier
  → success: persist result, then transition
  → explicit rejection: terminal failure
  → timeout/unknown: reconcile before retry
  → never map unknown to false success/failure
